# AI 문서 처리 프레임워크

> 공개소프트웨어 수업 팀 프로젝트

공공기관에 대량으로 접수되는 서류를 AI가 자동으로 분류하고 필드를 추출하는 로컬 문서 처리 프레임워크.

---

## 해결하는 문제

공공기관은 복지 신청, 대출 심사 등 다양한 업무에서 여러 종류의 서류를 대량으로 접수한다. 담당자가 서류를 일일이 열어 종류를 파악하고 필요한 항목을 확인하는 작업을 자동화한다.

- **문서 분류**: 제출된 서류가 어떤 종류인지 자동 판별
- **필드 추출**: 등록된 스키마 기준으로 필요한 항목값 추출
- **누락 리포트**: 추출되지 않은 항목을 담당자에게 알림

---

## 서비스 흐름

**관리자**: 모범 문서 업로드 → AI가 필드 목록 자동 추출 → 필요한 필드 선택 → 스키마 저장

**사용자**: 문서 이미지 제출 → 등록된 스키마 중 자동 분류 → 필드 추출 → 추출 결과 + 누락 필드 반환

```
[관리자] 모범 문서 업로드
    → PaddleOCR → 텍스트 추출
    → 한국어 텍스트 임베딩 → DB 저장
    → LLM → 필드 목록 자동 추출
    → 관리자가 필요한 필드 선택 → 스키마 저장

[사용자] 문서 이미지 제출
    → PaddleOCR → 텍스트 추출
    → 한국어 텍스트 임베딩 → 코사인 유사도 → Top-3 후보 스키마 선택
    → LLM(Qwen2.5-7B) → 최종 스키마 선택 + 필드 추출
    → 추출 결과 + 누락 필드 반환 → DB 저장
```

---

## 상용 서비스 대비 차별점

기존 IDP(Intelligent Document Processing) 서비스(AWS Textract, Azure Form Recognizer 등)는 새 문서 유형 추가 시 라벨링 데이터 수백 장과 재학습이 필요하다.

| | 상용 IDP | 본 프레임워크 |
|---|---|---|
| 스키마 등록 | 재학습 필요 | 모범 문서 1장 업로드로 즉시 등록 |
| 인프라 | 클라우드 API (데이터 외부 전송) | 완전 로컬 |
| 한국어 공문서 | 지원 미흡 | PaddleOCR 한국어 특화 |
| 비용 | 처리 건수당 과금 | 초기 서버 비용만 |

## 아키텍처

### 한국어 텍스트 임베딩 — 문서 분류

`klue/roberta-base` 모델로 OCR 텍스트를 768차원 벡터로 임베딩한다. 별도 학습 없이 코사인 유사도만으로 문서 유형을 분류한다.

한국 공공서류는 시각적 레이아웃이 유사하기 때문에 이미지 기반 임베딩보다 OCR 텍스트 기반 임베딩이 문서 유형 구분에 효과적이다.

- 모범 문서 등록 시 텍스트 임베딩을 DB에 저장
- 사용자 문서 제출 시 저장된 임베딩과 코사인 유사도 비교 → Top-3 후보 선택

### PaddleOCR — 텍스트 추출

한국어 특화 OCR. 문서 이미지의 텍스트를 추출하고 기울기를 자동 보정한다.

### Qwen2.5-7B — 필드 추출

4-bit QLoRA 양자화로 로컬 구동. LoRA 없이 순수 텍스트 입력만 사용한다.

- Top-3 후보 스키마 + OCR 텍스트를 함께 입력
- 최종 스키마 선택 + 필드 추출을 한 번에 처리

pip install -r requirements.txt

## 기술 스택

| 분류 | 기술 |
|---|---|
| 백엔드 | FastAPI, SQLAlchemy, SQLite |
| 문서 분류 | klue/roberta-base, 코사인 유사도 |
| OCR | PaddleOCR (한국어) |
| LLM | Qwen2.5-7B-Instruct, 4-bit QLoRA |
| 언어 | Python 3.11 |
